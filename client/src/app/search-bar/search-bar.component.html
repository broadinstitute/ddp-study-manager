How to search with the new filters?
How to use the search bar?
  • Click on the search bar to write your search query in it.
  • There is an algorithm that is constantly checking if the query being written in the search box is syntax-wise correct. Is it is wrong the font color will turn red and you can't search with that query.
  • Press the search button or the enter key to search!
  • You should use space between all the words and parenthesis.
    Example: Type ( a = 2 OR b = 3 ) instead of (a = 2 OR b = 3). In the second search statement the algorithm cannot distinguish between ( and a
  • To find the name of the fields to search for, select them in the customize view menu so they appear in the tissue list. When you hold your mouse on the column name in the table header, you can see their field names appear in little black pop ups. as an example when searching for the TypePX of a tissue, you can see find the field name when you hold your mouse on its header in the table, which will show you OD.typePX.
  • Use operators < , > , <= , >= , = , <> , LIKE
  • Use <> for not equal.
  • The LIKE operator is used when you want to see any results that contain that word
    For example, if you search for oD.typePX LIKE '%Biopsy%' you would get any description that has the word 'Biopsy' in them.
  • For the value following the LIKE you should always use single quotations '.
    Use % sign around the pattern you are looking for to get results that contain that pattern as well as the ones that are matched with it exactly.
    Example: oD.facility LIKE '%a' -> finds any values that end with "a"
    oD.facility LIKE 'a%' : Finds any values that starts with "a"
    oD.facility LIKE '%abcd%' Finds any values that have "abcd" in them.
  • Use single quotations ' to search for words and dates, but you don't need them for the numbers.
  • Your search statement MUST always start with AND.
  • If you are searching for an OR statement, you can easily do it after the first initial AND.
    Example: AND ( oD.typePX = 'Biopsy' OR oD.accessionNumber LIKE '%2%' )
  • The number of opening and closing parenthesis should match and should be valid.
    Example of a not valid one is ( ) ) or ) a OR b (.
  • If you are searching for empty values, use the phrase IS NULL and if you are searching for field that is not empty use IS NOT NULL
    Example: AND ( oD.typePX IS NULL AND t.smId IS NOT NULL )
  • When searching for additional realm specific fields, the search bar cannot be used. Use the filter menu on the left instead.
